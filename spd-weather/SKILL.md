# 🌤️ 智能天气查询技能 (SPD Weather Query)

## 触发条件
当用户询问天气、气温、降雨概率，或寻求穿衣、出行、洗车建议时，必须调用此技能。

## 城市名标准化（执行前必读）

脚本已内置后缀智能剥离（如"成都市" → "成都"、"甘孜藏族自治州" → "甘孜"）和省会映射（如"四川" → "成都"）。**LLM 只需处理以下两类脚本无法自动推断的情况：**

| 情形 | 处理规则 | 示例 |
|---|---|---|
| 网络别名 / 昵称 | 转换为地图标准名 | "魔都" → `上海`，"帝都" → `北京` |
| 区/县/乡镇级地名 | 上升到所属地市级 | "朝阳区" → `北京`，"西湖区" → `杭州` |

> **目标**：传入脚本的 `city_name` 应是**地市级**标准名称（如"绵阳"、"湘西"、"延边"）。含行政后缀、省名可直接传入，脚本会自动处理。

## 执行指令
在终端中执行以下命令，将 `<city_name>` 替换为标准化后的城市名：

```bash
uv run scripts/spd_weather.py "<city_name>"
```

## 🤖 返回结果处理与回复要求

### 核心表达原则（人性化与高效）

- **简明扼要，拒绝废话**：不要说类似“今天天气可真是个好日子呀”、“就像心情一样美滋滋的”这种无意义的寒暄。直接切入正题。
- **数据详实**：务必在口语化表达中包含**最高/最低气温、风力风向、降水情况**等核心气象数据，不要含糊其辞。
- **杜绝机器味**：不要把冰冷的数据列表直接粘贴给用户，翻译成自然、直接的大白话。
- **语气克制**：保持适当的温暖，适当使用 Emoji（如 ☀️ 🌧️ 🧣 🏃‍♂️ ☕）来丰富排版即可，不要过度热情或加戏。

### 强制包含的生活建议（场景化）

在简述完天气概况后，请结合具体的天气数据，主动为用户提供以下三个维度的生活建议：

#### 👕 穿衣与防护
结合气温和昼夜温差给出具体建议（例如："今天早晚温差有 10 度，建议采用'洋葱穿衣法'，带件薄风衣"、"今天紫外线较强，出门记得涂防晒霜、戴个墨镜哦😎"）。

#### 🚗 出行与日常
- 下雨/下雪：强制提醒带伞、提前出门防堵车、开车注意路滑
- 持续晴天：主动告知"未来几天都是大晴天，非常适合安排洗车或晾晒被子☀️"
- 大雾/大风：提醒减速慢行、远离高空坠物

#### 🏃 运动与休闲
- 适宜户外（温度舒适的晴天/多云）：强烈推荐去绿道晨跑、周末去郊外露营、飞盘、骑行或爬山
- 不宜户外（下雨、大风、极端高温或严寒）：建议将活动转入室内，例如"今天外面风雨交加，不如在室内做做瑜伽，或者去健身房挥洒汗水吧🧘‍♀️"

### 兜底处理

如果脚本返回了错误（如"获取失败"或"未找到城市"），请用幽默、抱歉的语气告知用户气象局的接口可能在开小差，并引导用户检查城市名称是否输入正确。
